  <action>
Create `DX.Blame.Hg.Blame.pas` with a dedicated template-based parser (completely separate from Git's porcelain parser per HGB-05).

**Interface section:**
- Uses: `DX.Blame.VCS.Types, DX.Blame.Hg.Types`
- Export procedure: `procedure ParseHgAnnotateOutput(const AOutput: string; var ALines: TArray<TBlameLineInfo>);`
- Export function: `function BuildAnnotateArgs(const ARelPath: string): string;`

**Template constant** (in implementation):
Per research recommendation, include `{desc|firstline}` as a 5th field for Summary:
```
cHgAnnotateTemplate = '{lines % ''{node}|{user}|{date|hgdate}|{lineno}|{desc|firstline}|{line}''}';
```

**BuildAnnotateArgs:**
Returns `'annotate -T "' + cHgAnnotateTemplate + '" "' + ARelPath + '"'`

**ParseHgAnnotateOutput implementation:**
- Split AOutput by #10 (LF)
- For each raw line, parse left-to-right with positional field extraction:
  1. Field 1 (positions 1-40): node hash. Verify char at position 41 is '|'
  2. Field 2: user -- from pos 42 to next '|'. Split "Name &lt;email&gt;" into Author/AuthorMail
  3. Field 3: hgdate -- "timestamp offset" format. Extract first token before space, convert via `StrToInt64Def` + `UnixToDateTime(LTimestamp, False)`
  4. Field 4: lineno -- extract integer, set both FinalLine and OriginalLine (Hg does not distinguish)
  5. Field 5: desc|firstline -- extract summary text up to next '|'
  6. Remainder after 5th '|': line content (ignored for blame)
- Skip lines shorter than 42 chars (hash + pipe minimum)
- Skip malformed lines (missing expected '|' at position 41)
- Detect uncommitted: `LInfo.CommitHash = cHgUncommittedHash` -> set IsUncommitted, override Author with cHgNotCommittedAuthor
- Trim trailing CR from each line before parsing
- Use `TList<TBlameLineInfo>` for accumulation, convert to array at end

IMPORTANT: Do NOT adapt or reference Git.Blame parser code. This is an independent parser per HGB-05.
IMPORTANT: Use Pos() with StartIndex overload for finding subsequent '|' delimiters, NOT Split('|') which would break on pipes in line content.

File MUST be UTF-8 with BOM and CRLF line endings.
