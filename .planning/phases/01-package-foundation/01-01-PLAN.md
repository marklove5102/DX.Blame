<task type="auto">
  <name>Task 2: Implement OTA registration unit with wizard, splash, about, and menu</name>
  <files>src/DX.Blame.Registration.pas</files>
  <action>
    Create `src/DX.Blame.Registration.pas` with XML doc header (MIT license, Copyright 2026 Olaf Monien). This is the central OTA lifecycle unit. UTF-8 with BOM, CRLF line endings.

    **Uses clause:** ToolsAPI, System.SysUtils, System.Classes, Vcl.Menus, Vcl.Graphics, Winapi.Windows, DX.Blame.Version.

    **Module-level variables (interface or implementation):**
    - GWizardIndex: Integer = -1 (wizard registration index)
    - GAboutPluginIndex: Integer = -1 (about box registration index)
    - GMenuParentItem: TMenuItem = nil (root menu item for cleanup)

    **TDXBlameWizard class** (implementation section):
    - Inherits from TNotifierObject, implements IOTAWizard
    - GetIDString returns 'DX.Blame'
    - GetName returns cDXBlameName (from Version unit)
    - GetState returns [wsEnabled]
    - Execute is a no-op (Phase 1 -- wizard exists for IDE registration only)

    **CreateToolsMenu procedure** (implementation):
    - Get INTAServices from BorlandIDEServices (exit if not supported)
    - Create parent TMenuItem: Caption='DX Blame', Name='DXBlameMenu'
    - Create child TMenuItem 'Enable Blame': Name='DXBlameEnableItem', Enabled=False
    - Create child TMenuItem 'Settings...': Name='DXBlameSettingsItem', Enabled=False
    - Add children to parent, add parent via LNTAServices.AddActionMenu('ToolsMenu', nil, GMenuParentItem)
    - Store parent in GMenuParentItem for cleanup

    **RemoveToolsMenu procedure** (implementation):
    - FreeAndNil(GMenuParentItem) -- children freed by owner

    **Register procedure** (interface section, exported):
    - Get IOTAWizardServices, call AddWizard(TDXBlameWizard.Create), store in GWizardIndex
    - Get IOTAAboutBoxServices, call AddPluginInfo with:
      - Title: cDXBlameName
      - Description: cDXBlameDescription + sLineBreak + cDXBlameCopyright
      - Bitmap: LoadBitmap(FindResourceHInstance(HInstance), 'DXBLAMESPLASH') -- do NOT free this bitmap (IDE takes ownership per RESEARCH.md Pitfall 2)
      - IsUnregistered: False
      - LicenseStatus: '' (empty)
      - SKUName: cDXBlameVersion
    - Store result in GAboutPluginIndex
    - Call CreateToolsMenu

    **initialization section:**
    - Register splash screen: check `if Assigned(SplashScreenServices)` then call AddPluginBitmap with:
      - PluginName: cDXBlameName
      - Bitmap: LoadBitmap(FindResourceHInstance(HInstance), 'DXBLAMESPLASH') -- this bitmap CAN be freed after registration (splash copies it) but simplest to let it leak (tiny 24x24)
      - IsUnregistered: False
      - LicenseStatus: 'Open Source'
    - IMPORTANT: Use AddPluginBitmap, NOT AddProductBitmap (broken per QC 42320, see RESEARCH.md anti-patterns)
    - IMPORTANT: Do NOT use `BorlandIDEServices as IOTASplashScreenServices` -- use global SplashScreenServices variable with Assigned check

    **finalization section** (reverse order per RESEARCH.md Pattern 5):
    1. RemoveToolsMenu (UI elements first)
    2. If GWizardIndex > 0 then remove wizard via (BorlandIDEServices as IOTAWizardServices).RemoveWizard
    3. If GAboutPluginIndex > 0 then remove about box via (BorlandIDEServices as IOTAAboutBoxServices).RemovePluginInfo

    **Anti-patterns to avoid** (from RESEARCH.md):
    - Do NOT hardcode menu indices to find Tools menu
    - Do NOT free the about box bitmap
    - Do NOT use AddProductBitmap
    - Do NOT register splash in Register procedure (must be in initialization)
    - Do NOT split OTA lifecycle across multiple units
  </action>
