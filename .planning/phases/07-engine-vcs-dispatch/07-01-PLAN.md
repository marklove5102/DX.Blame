<objective>
Refactor DX.Blame.Engine and DX.Blame.CommitDetail to dispatch all VCS operations through IVCSProvider instead of direct Git unit calls.

Purpose: Engine and CommitDetail are the core dispatch layer -- all blame execution and commit detail fetching flows through them. Making them provider-agnostic is the prerequisite for Navigation and Popup migration in Plan 02.
Output: Engine with FProvider field, VCSAvailable property, provider-injected threads; CommitDetail with provider-parameterized async fetch.
</objective>

  <name>Task 1: Refactor Engine to use IVCSProvider</name>
  <files>src/DX.Blame.Engine.pas</files>
  <action>
Refactor DX.Blame.Engine to own and dispatch through IVCSProvider:

**Private fields -- rename and add:**
- `FGitPath: string` -> REMOVE (provider handles executable internally)
- `FGitAvailable: Boolean` -> rename to `FVCSAvailable: Boolean`
- `FGitNotified: Boolean` -> rename to `FVCSNotified: Boolean`
- ADD `FProvider: IVCSProvider`

**Public properties -- rename:**
- `GitAvailable: Boolean read FGitAvailable` -> `VCSAvailable: Boolean read FVCSAvailable`
- ADD `property Provider: IVCSProvider read FProvider`
- Keep `RepoRoot` as-is

**Constructor/Destructor:**
- In Create: change `FGitAvailable := False` to `FVCSAvailable := False`, same for FVCSNotified
- Destructor: no explicit FProvider.Free needed (interface reference counting handles it)

**Initialize method:**
- Create provider: `FProvider := TGitProvider.Create;` (hardcoded for now, Phase 8 adds discovery)
- Replace `FGitPath := FindGitExecutable` with `LExePath := FProvider.FindExecutable`
- Replace `FRepoRoot := FindGitRepoRoot(AProjectPath)` with `FRepoRoot := FProvider.FindRepoRoot(AProjectPath)`
- Replace all `FGitAvailable` references with `FVCSAvailable`
- Replace all `FGitNotified` references with `FVCSNotified`
- Update log message: 'DX.Blame: VCS executable not found' instead of 'git not found'

**TBlameThread:**
- Constructor: change from `(AEngine: TBlameEngine; const AGitPath, ARepoRoot, AFileName: string)` to `(AEngine: TBlameEngine; AProvider: IVCSProvider; const ARepoRoot, AFileName: string)`
- Field: replace `FGitPath: string` with `FProvider: IVCSProvider` (interface-typed for correct ref counting)
- Execute: replace `TGitProcess.Create(FGitPath, FRepoRoot)` + manual command building with:
  - `LExitCode := FProvider.ExecuteBlame(FRepoRoot, FFileName, LOutput, FProcessHandle)`
  - Replace `ParseBlameOutput(LOutput, LLines)` with `LLines := FProvider.ParseBlameOutput(LOutput)`
  - Remove the LProcess local var and try/finally block
  - Remove the LRelPath computation (provider handles path normalization)
- Cancel: replace `TGitProcess.CancelProcess(FProcessHandle)` with `TVCSProcess.CancelProcess(FProcessHandle)`

**RequestBlame method:**
- Replace `FGitAvailable` check with `FVCSAvailable`
- Replace `TBlameThread.Create(Self, FGitPath, FRepoRoot, AFileName)` with `TBlameThread.Create(Self, FProvider, FRepoRoot, AFileName)`

**RequestBlameDebounced:**
- Replace `FGitAvailable` with `FVCSAvailable`

**OnProjectSwitch:**
- Replace `ClearDiscoveryCache` (direct Git.Discovery call) with `if FProvider <> nil then FProvider.ClearDiscoveryCache`
- Set `FProvider := nil` before calling Initialize (which creates a new provider)

**Log messages:**
- Update debug messages from 'git' to 'VCS' where appropriate (e.g., 'skip blame (VCS not available)')

**Uses clause changes (implementation section):**
- REMOVE: `DX.Blame.Git.Discovery`, `DX.Blame.Git.Process`, `DX.Blame.Git.Blame`
- ADD: `DX.Blame.VCS.Provider`, `DX.Blame.VCS.Process`, `DX.Blame.Git.Provider`
- KEEP: `ToolsAPI`, `ToolsAPI.Editor`, `DX.Blame.CommitDetail`

**Forward declaration:**
- Keep `TBlameThread = class;` forward declaration in interface section
- Update the full TBlameThread declaration in interface section to match new constructor signature
